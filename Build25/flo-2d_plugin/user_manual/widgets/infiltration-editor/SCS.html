SCS

Global Uniform Infiltration

The SCS infiltration editor can add global or spatially variable infiltration data to the INFIL.DAT file for infiltration curve numbers.

  1. Set up the Global Infiltration first. Click Global Infiltration.

  1. Fill the Global Infiltration dialog box.

  1. The INFIL.DAT file looks like this:

  2. Where the infiltration type is 2 = SCS infiltration.

  3. The 85 is the uniform curve number for each grid.

  4. The 0.01 is the initial abstraction.

Spatial SCS Infiltration from Infiltration Areas User Layer

Note

This method is the most effective way to sample SCS data. If using the other calculators, review SCS column for errors.

Spatially variable floodplain infiltration is set by digitizing infiltration polygons or importing infiltration polygons. Use the polygon editor to digitize spatially variable infiltration. Create a polygon to represent an area of infiltration.

  1. Select the Infiltration Areas user layer.

  2. Click the editor pencil and snapping magnet button.

  1. Create the polygons the represent areas with the same curve number.

  2. Fill the table for the infiltration data.

  1. The finished table has a CN for every polygon.

  2. Click the Save button to save the attributes.

  3. Click the pencil button to close the editor.

  1. Check the data in the Infiltration Editor Widget. Click the Schematize button to complete the process.

  1. The spatially variable INFIL.DAT looks like this:

Curve Number Generator

  1. If necessary, add the Plugin Curve Number Generator.

  1. Open the Curve Number Generator.

  1. Set the Area Boundary to the Grid. Check the boxes and click OK.

  1. Click Close when process is finished. The Curve Number Polygon Layer can be used in the next section.

SCS Calculator From Single Shapefile

Warning

If applying this method, review min and max of the SCS field. This method only works on polygon shapefiles that have no geometric deficiencies. If this method results in errors, copy the polygons to the User layer field and use the User Layer Method.

This option will add spatially variable infiltration data to the grid from a shapefile with one CN attribute field.

  1. Click the Calculate SCS CN button.

  1. Check the ‘Assign SCS Curve Number from Polygon Layer - single field’.

  2. Select the layer and field with the infiltration data.

  3. This method works for shapefiles that have a CN already calculated.

  4. Click OK to calculate a spatially variable CN value for every grid element.

  1. When the calculation is complete, the following box will appear. Click OK to close the box.

SCS Calculator From Single Shapefile Multiple Fields Pima County Method

Use this option for Pima County to calculate SCS curve number data from a single layer with multiple fields. This is a vector layer with polygon features and field to define the landuse/soil group, vegetation coverage and impervious space. This option was developed specifically for Pima County.

The data should be arranged as shown in the attribute table.

  1. Click the Calculate SCS CN button.

  1. Check the ‘Assign SCS Curve Number from Polygon Layer - multiple fields’.

  2. Select the layer and fields with the infiltration data and click OK to run the calculator.

  1. When the calculation is complete, the following box will appear. Click OK to close the box.

SCS Calculator From Raster Layer

This option will add spatially variable infiltration data to the grid from a raster with cells containing CN values. Important properties:

Important

  • The raster must have the same coordinate reference system (CRS) as the project. If the CRS is missing or is set by the user, save the raster with the correct CRS.

  • The best resolution of the grid element CN is achieved when the CN raster pixel size is smaller than the grid element size.

  • The raster warp method uses a weighted average to warp the original raster pixels to the cell size pixels.

  1. Click the Calculate SCS CN button.

  1. Check the ‘Assign SCS Curve Number from Raster Layer’.

  2. Select the raster containing CN values from the dropdown or choose a raster from the file dialog.

  3. Set the NODATA value.

  4. Select the resampling algorithm.

  5. Select the Fill NODATA option to set the CN of empty grid elements from neighbors. This is only necessary if there are empty raster pixels.

  6. Select the multithread option to use all CPU’s for running the algorithm.

  7. Click OK to calculate a spatially variable CN value for every grid element.

  1. When the calculation is complete, the following box will appear. Click OK to close the box.

  1. The INFIL.DAT file looks like this.

Troubleshooting

  1. Infiltration calculators all use intersection tools. This can cause problems if the shapefiles are not set up correctly. Specifically, land use and soils shapefiles that may have been converted from raster data. If errors persist, try “fix geometry”, “simplify”, and “dissolve” on the source shapefiles. These tools are part of the QGIS Processing Toolbox. They can also be corrected in ArcGIS if the datasets are very large.

  2. Make sure the shapefiles completely cover the grid. If a grid element is outside the coverage of the infiltration, QGIS will show an error.

  3. Make sure the shapefile fields have a correctly defined number type. The shapefiles that are supplied with the QGIS Lessons will help define the Field Variable Format such as string, whole number or decimal number.
